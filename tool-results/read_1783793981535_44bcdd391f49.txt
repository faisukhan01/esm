     1→# eSM — Electronic School Management System — Worklog
     2→
     3→## Project Overview
     4→Building a complete, modern School Management System (eSM) based on the "eSM Modules Detail.pdf".
     5→- **Frontend**: Next.js 16 + Three.js (3D hero) + shadcn/ui + Tailwind CSS 4
     6→- **Backend**: Node.js + Express.js mini-service (port 3001) accessed via gateway `?XTransformPort=3001`
     7→- **Database**: In-memory seed data inside the Express service (fast, no migration friction) + Prisma available for persistence
     8→- **Purpose**: Educational, clean, eye-catching, highly competitive with USA school portals. No .apk — pure web stack.
     9→

    40→## Architecture Decisions
    41→- Single SPA on `/` (the only user-visible route). Client-side state switches between Landing → Login → Dashboard.
    42→- Dashboard uses a left sidebar listing all modules; content area renders the selected module.
    43→- Express backend serves JSON for: dashboard stats, students, attendance, fees, results, academics, sms, library, transport, hr, events, finance.
    44→- Three.js animated 3D hero on the landing page (graduation cap / particles / floating geometric campus).
    45→- Color theme: deep emerald + warm gold accent (academic prestige, not the default indigo/blue), with clean neutrals.
