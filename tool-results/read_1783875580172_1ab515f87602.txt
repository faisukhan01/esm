     1‚Üí# eSM ‚Äî Electronic School Management System ‚Äî Worklog

     3‚Üí## Project Overview
     4‚ÜíBuilding a complete, modern School Management System (eSM) based on the "eSM Modules Detail.pdf".
     5‚Üí- **Frontend**: Next.js 16 + Three.js (3D hero) + shadcn/ui + Tailwind CSS 4
     6‚Üí- **Backend**: Node.js + Express.js mini-service (port 3001) accessed via gateway `?XTransformPort=3001`
     7‚Üí- **Database**: In-memory seed data inside the Express service (fast, no migration friction) + Prisma available for persistence
     8‚Üí- **Purpose**: Educational, clean, eye-catching, highly competitive with USA school portals. No .apk ‚Äî pure web stack.
     9‚Üí

    40‚Üí## Architecture Decisions
    41‚Üí- Single SPA on `/` (the only user-visible route). Client-side state switches between Landing ‚Üí Login ‚Üí Dashboard.
    42‚Üí- Dashboard uses a left sidebar listing all modules; content area renders the selected module.
    43‚Üí- Express backend serves JSON for: dashboard stats, students, attendance, fees, results, academics, sms, library, transport, hr, events, finance.
    44‚Üí- Three.js animated 3D hero on the landing page (graduation cap / particles / floating geometric campus).
    45‚Üí- Color theme: deep emerald + warm gold accent (academic prestige, not the default indigo/blue), with clean neutrals.
